  <name>Task 1: Add process crash handlers and error handling to GET/DELETE routes</name>
  <files>src/server.ts</files>
  <action>
Add `unhandledRejection` and `uncaughtException` handlers at the TOP of `main()` (before any async work), so they catch errors from any part of startup or runtime:

```typescript
process.on("unhandledRejection", (reason, promise) => {
  logger.error("Unhandled rejection at:", promise, "reason:", reason);
  process.exit(1);
});

process.on("uncaughtException", (error) => {
  logger.error("Uncaught exception:", error);
  process.exit(1);
});
```

Place these BEFORE the `logger.info("v${config.version} starting...")` line so even startup crashes are caught.

Wrap the GET /mcp handler body in try/catch:

```typescript
app.get("/mcp", async (req, res) => {
  const sessionId = req.headers["mcp-session-id"] as string | undefined;
  if (!sessionId || !transports[sessionId]) {
    res.status(400).send("Invalid or missing session ID");
    return;
  }
  try {
    await transports[sessionId].handleRequest(req, res);
  } catch (error) {
    logger.error("MCP SSE error:", error);
    if (!res.headersSent) {
      res.status(500).send("Internal server error");
    }
  }
});
```

Wrap the DELETE /mcp handler body in try/catch:

```typescript
app.delete("/mcp", async (req, res) => {
  const sessionId = req.headers["mcp-session-id"] as string | undefined;
  if (!sessionId || !transports[sessionId]) {
    res.status(400).send("Invalid or missing session ID");
    return;
  }
  try {
    await transports[sessionId].handleRequest(req, res);
  } catch (error) {
    logger.error("MCP session teardown error:", error);
    if (!res.headersSent) {
      res.status(500).send("Internal server error");
    }
  }
});
```

<task type="auto">

  <files>src/db/index.ts, docker-compose.prod.yml</files>
  <action>
In `src/db/index.ts`, change `idle_timeout: 20` to `idle_timeout: 0` (disables idle timeout — connections stay open for the lifetime of the pool). This is appropriate for a long-running server with bursty MCP traffic. Also add `connect_timeout: 10` for explicit connection timeout:

```typescript
const client = postgres(databaseUrl, {
  max: 10,
  idle_timeout: 0,
  connect_timeout: 10,
  onnotice: () => {},
});
```

In `docker-compose.prod.yml`, add `restart: unless-stopped` to the `agent-brain` service (at the same level as `build:`, `ports:`, etc.):

```yaml
services:
  agent-brain:
    build: .
    restart: unless-stopped
    ports: ...
```
