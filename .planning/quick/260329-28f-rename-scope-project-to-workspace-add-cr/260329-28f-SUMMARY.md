# Quick Task 260329-28f: Rename scope project to workspace, add cross-workspace project scope Summary

Three-scope enum (workspace/user/project) with migration renaming existing data, cross-workspace project scope visible everywhere, and agent-auto rejection for project scope.

## What Changed

### Schema and Types

- `MemoryScope` type: `"workspace" | "user" | "project"` (was `"project" | "user"`)
- DB enum `memory_scope` has three values: workspace, user, project
- `memories.project_id` is now nullable (project-scoped memories don't need one)
- `memories.scope` defaults to `"workspace"` (was `"project"`)
- `MemoryCreate.project_id` is optional

### Migration (0003_rename_project_to_workspace.sql)

- Adds `workspace` to the enum
- Updates all existing `project` rows to `workspace`
- Repurposes the existing `project` enum value for cross-workspace semantics
- Makes `project_id` nullable and changes default scope

### Repository Layer

- `search()`: All modes include project-scoped memories via OR condition
- `list()`: New `"project"` scope option filters by `scope=project` only (no project_id filter)
- `listRecentBothScopes()`: Includes project-scoped memories
- `findRecentActivity()`: Includes workspace and project scopes
- `findDuplicates()`: Project-scope dedup checks all project-scoped memories globally

### Service Layer

- `canAccess()`: Both workspace and project scopes grant shared access
- `create()`: Guard 0 rejects `scope=project` + `source=agent-auto` with ValidationError
- `create()`: Default scope is now `"workspace"`
- `search()`: Signature accepts `"workspace" | "user" | "both"`
- `getWithComments()`: `can_comment` works for both workspace and project shared scopes

### Tool Layer

- `memory_create`: scope enum `["workspace", "user", "project"]`, project_id optional
- `memory_search`: scope enum `["workspace", "user", "both"]`
- `memory_list`: scope enum `["workspace", "user", "project"]`, project_id optional

### Prompts

- Memory guidance documents all three scopes with usage guidance

### Tests

- All 24 test files updated from old `scope: "project"` to `scope: "workspace"`
- 5 new tests added for project scope behavior:
  - Cross-workspace visibility
  - Agent-auto rejection (ValidationError)
  - Manual creation succeeds
  - Search scope=both includes all three scopes
  - Default scope is workspace
- 152 tests pass, 0 failures
