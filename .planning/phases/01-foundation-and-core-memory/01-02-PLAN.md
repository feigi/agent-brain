   - `embed(text)` generates a deterministic 512-element vector using a hash of the input text:
     ```typescript
     async embed(text: string): Promise<number[]> {
       let hash = 0;
       for (let i = 0; i < text.length; i++) {
         hash = ((hash << 5) - hash + text.charCodeAt(i)) | 0;
       }
       const vector = Array.from({ length: 512 }, (_, i) => {
         const val = Math.sin(hash * (i + 1) * 0.001);
         return Number(val.toFixed(6));
       });
       return vector;
     }
     ```
   - Key property: same input always produces same vector (deterministic for tests)
   - Key property: different inputs produce different vectors (useful for search testing)

3. Create src/repositories/memory-repository.ts -- Drizzle/pgvector implementation:
- Class `DrizzleMemoryRepository` implements `MemoryRepository`
- Constructor takes `Database`
- `create(memory)`: INSERT into memories table. The `memory` param includes the embedding vector. Return the inserted row (without embedding vector in the returned Memory type).
- `findById(id)`: SELECT by id WHERE archived_at IS NULL. Return Memory or null. Do NOT return the embedding vector (D-44).
- `update(id, expectedVersion, updates)`: UPDATE with WHERE id = id AND version = expectedVersion. Increment version by 1. Set updated_at to now(). If `updates.embedding` is provided, update embedding + embedding_model + embedding_dimensions. If result is empty array, throw `ConflictError` per D-30 / RESEARCH Pattern 7. Return updated Memory.
- `archive(ids)`: UPDATE memories SET archived_at = now(), embedding = NULL WHERE id IN (ids) AND archived_at IS NULL. Per D-28, null out the embedding vector. Per D-67, archiving already-archived memories is idempotent -- return count of newly archived (rows where archived_at was NULL).
- `search(options)`: Per RESEARCH Pattern 5. Compute `similarity = 1 - cosineDistance(memories.embedding, options.embedding)`. WHERE conditions:
  - If scope = "project": `project_id = options.project_id` (SCOP-01, D-47)
  - If scope = "user": `author = options.user_id` AND `scope = 'user'` (SCOP-02)
  - `archived_at IS NULL`
  - `similarity > options.min_similarity` (default 0.3, per D-42)
  - ORDER BY similarity DESC, LIMIT options.limit (default 10, per D-41)
  - Return MemoryWithScore[] (full memory object + similarity score, per D-43, without embedding vector per D-44)
- `list(options)`: Per RESEARCH Pattern 6 (cursor-based pagination, D-46).
  - WHERE: project_id match (scope handling same as search), archived_at IS NULL
  - Optional filters: type (D-48), tags (D-48) using array overlap
  - Sort by options.sort_by (default created_at) in options.order (default desc) per D-45
  - Cursor-based pagination: fetch limit+1 rows to determine has_more
  - Return { memories, has_more, cursor (from last returned row) }
- `findStale(options)`: Per D-12. SELECT memories WHERE project_id = options.project_id AND archived_at IS NULL AND (verified_at IS NULL OR verified_at < now() - threshold_days). Same cursor pagination pattern. Return { memories, has_more, cursor }.
- `verify(id)`: UPDATE memories SET verified_at = now(), updated_at = now() WHERE id = id AND archived_at IS NULL. Return updated Memory or null if not found.

IMPORTANT: In all SELECT queries that return Memory objects, do NOT include the `embedding` column in the select list (D-44). Use explicit column selection, not `SELECT *`.

4. Create src/services/memory-service.ts -- orchestrates embedding + storage:
- Class `MemoryService`
- Constructor takes `MemoryRepository`, `ProjectRepository`, `EmbeddingProvider`
- `create(input: MemoryCreate)`:
  1. Validate content length -- if > 4000 chars, log warning but allow (D-20)
  2. Auto-generate title if not provided: first 80 chars of content + "..." (D-03)
  3. Call `projectRepository.findOrCreate(input.project_id)` to ensure project exists (D-34)
  4. Generate embedding: concatenate title + "\n\n" + content (D-19), call `embeddingProvider.embed()`. If embedding fails, throw EmbeddingError (D-54 -- fail the save, no partial state)
  5. Generate id via `generateId()`
  6. Insert via `memoryRepository.create()` with embedding, embedding_model, embedding_dimensions from provider
  7. Return Envelope<Memory> with timing in meta
- `get(id: string)`:
  1. Call `memoryRepository.findById(id)`
  2. If null, throw NotFoundError
  3. Return Envelope<Memory>
- `update(id: string, expectedVersion: number, updates: MemoryUpdate)`:
  1. If content or title changed, re-embed: concatenate (updates.title ?? existing.title) + "\n\n" + (updates.content ?? existing.content), call embeddingProvider.embed() (D-27)
  2. To get existing title/content for re-embedding when only one changes, first call findById
  3. Update via `memoryRepository.update()` with new embedding if applicable
  4. Return Envelope<Memory>
- `archive(ids: string | string[])`:
  1. Normalize to array (D-06: accepts single ID or array)
  2. Call `memoryRepository.archive(ids)`
  3. Return Envelope<{ archived_count: number }>
- `search(query: string, project_id: string, scope: "project" | "user", user_id?: string, limit?: number, min_similarity?: number)`:
  1. Generate embedding for query text via embeddingProvider.embed()
  2. Call `memoryRepository.search()` with embedding and options
  3. Return Envelope<MemoryWithScore[]> with count in meta
- `list(options: ListOptions)`:
  1. Call `memoryRepository.list(options)`
  2. Return Envelope<Memory[]> with count, has_more, cursor in meta
- `verify(id: string)`:
  1. Call `memoryRepository.verify(id)`
  2. If null, throw NotFoundError
  3. Return Envelope<Memory>
- `listStale(project_id: string, threshold_days: number, limit?: number, cursor?: ...)`:
  1. Call `memoryRepository.findStale()`
  2. Return Envelope<Memory[]> with count, has_more, cursor in meta

All methods measure timing (Date.now() before and after) and include it in meta.timing.
