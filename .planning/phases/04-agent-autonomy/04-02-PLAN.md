**docs/hooks/memory-session-review.sh** -- Stop hook script (per D-07). MUST check `stop_hook_active` to prevent infinite loops (Pitfall 2 from RESEARCH):

```bash
#!/bin/bash
# Claude Code Stop Hook: Session-End Memory Review
# Triggers a memory review when Claude is about to stop responding.
# Install: copy to .claude/hooks/ and add to .claude/settings.json

INPUT=$(cat)
STOP_ACTIVE=$(echo "$INPUT" | jq -r '.stop_hook_active // "false"')

# Prevent infinite loop -- if already in a stop hook cycle, let Claude stop
if [ "$STOP_ACTIVE" = "true" ]; then
  exit 0
fi

# Block the stop and ask Claude to review session learnings
cat <<'REVIEW'
{
  "decision": "block",
  "reason": "Before ending, please perform a session-end memory review: reflect on this session's work and save any key learnings, decisions, conventions, or patterns worth remembering using memory_create with source 'session-review'. Focus on insights that will help in future sessions. Then you may stop."
}
REVIEW
exit 0
```

**docs/hooks/README.md** -- Setup instructions covering:

1. Prerequisites: Claude Code with hooks support, jq installed
2. Installation steps:
   - Copy `memory-session-review.sh` to `.claude/hooks/` in your project
   - Make executable: `chmod +x .claude/hooks/memory-session-review.sh`
   - Add the Stop hook configuration from `settings-snippet.json` to your `.claude/settings.json`
3. How it works: Claude Code fires the Stop hook when Claude is about to stop. The hook blocks the first stop attempt and asks Claude to perform a memory review. On the second attempt (`stop_hook_active: true`), it lets Claude stop normally.
4. Note per D-09: hooks are optional enhancements for Claude Code users. Other MCP clients use the natural-breakpoints pattern from the `memory-guidance` prompt.
